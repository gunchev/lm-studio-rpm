# Agent Instructions for lm-studio-rpm

This repository converts LM Studio Debian packages to RPM format for Fedora/RHEL/CentOS/SUSE systems.

## Project Structure

| File | Purpose |
|------|---------|
| `gen_spec.py` | Python script that extracts metadata from `.deb` packages and generates RPM spec files |
| `lm-studio.spec` | RPM spec file (auto-generated by `gen_spec.py`) |
| `Makefile` | Build automation with phony targets |
| `go.sh` | Legacy manual build script (not maintained) |
| `*.deb` / `*.rpm` | Package files (input/output) |

## Build Commands

### Using Make (Recommended)

```bash
make all              # Generate spec and build RPM (full pipeline)
make spec             # Generate spec file only
make rpm              # Build RPM from existing spec (skips spec regeneration)
make test             # Test metadata extraction (./gen_spec.py output to stdout)
make clean            # Remove build artifacts (*.rpm, lm-studio-buildroot/, lm-studio/)
make install-deps     # Install required build dependencies
make help             # Display available targets and variables
```

### Direct Commands

```bash
# Generate spec file
./gen_spec.py "LM-Studio-VERSION-x64.deb" -o lm-studio.spec

# Extract metadata for testing (output to stdout)
./gen_spec.py "LM-Studio-VERSION-x64.deb" -o /dev/stdout

# Build RPM manually
mkdir -p buildroot && dpkg-deb -x package.deb buildroot
rpmbuild --buildroot="$(pwd)/buildroot" -bb --target x86_64 lm-studio.spec
```

### Variables

| Variable | Description | Default |
|----------|-------------|---------|
| `DEB` | Path to deb package | First `LM-Studio-*-x64.deb` found |
| `SPEC` | Output spec file | `lm-studio.spec` |
| `RPM_DIR` | RPM output directory | `.` |
| `BUILD_ROOT` | Build root directory | `lm-studio-buildroot` |

## Code Style

### Python (gen_spec.py)

| Aspect | Convention |
|--------|------------|
| **Shebang** | `#!/usr/bin/env python3` |
| **Python version** | Python 3 only |
| **Type hints** | Required for function parameters and return types |
| **Line length** | 100 characters maximum |
| **Indentation** | 4 spaces (no tabs) |
| **String quotes** | Double quotes `"` for strings; single quotes `'` only when nested |

**Import order** (alphabetical within each group):
1. Standard library modules (`argparse`, `os`, `re`, `subprocess`, etc.)
2. Third-party modules (none in this project)
3. Local modules (none in this project)

**Naming conventions**:
- Functions: `snake_case` (e.g., `extract_deb_control`)
- Classes: `PascalCase` (not used in this project)
- Constants: `SCREAMING_SNAKE_CASE` (e.g., `DEFAULT_SPEC_NAME`)
- Variables: `snake_case` (e.g., `deb_path`, `control_data`)

**Error handling patterns**:
```python
# Fatal errors - print to stderr and exit
print(f"Error message: {result.stderr}", file=sys.stderr)
sys.exit(1)

# Subprocess calls with error checking
result = subprocess.run(["command", "arg"], capture_output=True, text=True)
if result.returncode != 0:
    print(f"Error: {result.stderr}", file=sys.stderr)
    sys.exit(1)

# Optional operations (should not fail build)
command || true
```

### Shell Scripts (go.sh, %post, %preun sections)

| Aspect | Convention |
|--------|------------|
| **Shebang** | `#!/usr/bin/env bash` |
| **Variables** | Use `${var}` syntax, always double-quote expansions |
| **Conditionals** | Prefer `test` over `[ ]` for clarity |
| **Error handling** | Use `|| true` for optional commands |
| **Checks** | Use `command -v foo >/dev/null 2>&1` to check command existence |

**Example patterns**:
```bash
# Variable expansion with quoting
rm -rf "${BUILD_ROOT}"

# Existence checks
if test -d "${BUILD_ROOT}/usr/share/icons/hicolor/0x0"; then
    mv "${BUILD_ROOT}/usr/share/icons/hicolor/0x0" \
       "${BUILD_ROOT}/usr/share/icons/hicolor/1024x1024"
fi

# Optional commands (should not fail)
chmod 4755 '/opt/LM-Studio/chrome-sandbox' || true
```

### RPM Spec Files

**Section order**:
```
Name → Version → Release → Summary → License → Vendor → URL → 
BuildArch → %description → %post → %postun → %files → %changelog
```

**Macro usage**: Use RPM macros for portability:
- `%{NAME}`, `%{VERSION}`, `%{RELEASE}`, `%{ARCH}`
- `%{_rpmdir}`, `%{_rpmfilename}`, `%{_unpackaged_files_terminate_build}`

**Comments**: Use `##` prefix for extracted metadata markers from dpkg-deb.

**%postun section**: Handles package removal - removes the `/usr/bin/lm-studio` symlink via `update-alternatives` or direct removal. This script runs when the RPM is uninstalled.

**Example postrm script from deb**:
```bash
#!/bin/bash

# Delete the link to the binary
if type update-alternatives >/dev/null 2>&1; then
    update-alternatives --remove 'lm-studio' '/usr/bin/lm-studio'
else
    rm -f '/usr/bin/lm-studio'
fi
```

## Workflow

1. Download a new LM Studio `.deb` package from https://lmstudio.ai/download
2. Run `make test` to verify metadata extraction works
3. Run `make spec` to regenerate the spec file
4. Review `lm-studio.spec` for correctness
5. Run `make rpm` to build the RPM package
6. Test the RPM in a container/VM: `rpm -ivh lm-studio-*.x86_64.rpm`
7. Verify installation: `rpm -qi lm-studio`

## Troubleshooting

### Common Issues

**No deb files found**: Ensure you're in the correct directory and a `.deb` file exists:
```bash
ls -la LM-Studio-*.deb
```

**RPM build fails with unpackaged files**: The `_unpackaged_files_terminate_build` macro is set to 0 to allow this, but verify `%files` section lists all needed paths.

**Icon directory issue**: The build script automatically fixes the `0x0` icon directory (should be `1024x1024`).

**Permission denied**: Ensure write permissions to the output directory.

### Debug Commands

```bash
# Inspect deb package contents
dpkg-deb --contents LM-Studio-*.deb

# View deb control information
dpkg-deb --info LM-Studio-*.deb

# View postinst script
dpkg-deb --info LM-Studio-*.deb postinst

# Check generated spec
cat lm-studio.spec
```
